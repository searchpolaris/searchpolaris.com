ChatGPT의 검색 시스템은 단순한 RAG가 아닙니다. 확률 분류기, 쿼리 팬아웃, 제3자 데이터 파이프라인, 엔티티 연결이 결합된 복합 시스템입니다.

Sonic 분류기

검색 전, 확률 분류기가 외부 데이터 필요 여부를 판단합니다. search_prob이 임계값(65%)을 넘으면 웹 검색이 실행됩니다. "프랑스 수도"는 낮은 확률, "어제 챔피언스리그 결과"는 높은 확률로 판정됩니다.

196ms의 지연시간으로 작동합니다.

{  "sonic_classification_result": {    "latency_ms": 196.58,    "search_prob": 0.7135,    "force_search_threshold": 0.65,    "classifier_config_name": "sonic_force_pg_switcher"  }}

팬아웃 엔진

사용자 질문 1개가 N개의 병렬 쿼리로 변환됩니다. 검색(13개, 사고모드 20+), 쇼핑(짧은 제품 중심 쿼리), 이미지(38+개) 팬아웃이 동시에 작동합니다.

대부분 2가지 유형이 조합됩니다.

데이터 제공자 생태계

OpenAI는 수십억을 투자했지만 핵심 전략은 제3자를 통한 구글 스크래핑입니다. SerpAPI로 구글 검색, SearchApi.io로 구글 쇼핑, Getty로 이미지, 'b1' 프로바이더로 구글 지도 데이터를 가져옵니다.

2026년 1월 구글이 SerpAPI를 고소했습니다. 이는 chatGPT가 제 3자 API를 활발하게 사용함을 증명합니다.

엔티티 연결 시스템

ChatGPT는 자체 NER로 인물, 기업, 장소 등을 감지하고 사이드바를 생성합니다. gpt-5-instant 모델이 동적 프롬프트로 맥락 정보를 만듭니다.

코드에서 'Hide Ads' 문구가 발견되어 향후 광고 시스템 준비 가능성이 제기됩니다.
