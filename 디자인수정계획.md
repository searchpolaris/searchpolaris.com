# SearchPolaris index.html 디자인 통일 & 텍스트 수정 계획

## Context
SearchPolaris 메인 페이지(index.html)는 9개 섹션을 각기 다른 사람이 작업하여 디자인 요소(선 스타일, 타이포그래피, 뱃지 등)가 섹션마다 불일치함. 이를 통일된 디자인 시스템으로 정리하고, FAQ 항목 삭제 및 탭 애니메이션 순서 수정을 진행.

**파일**: `index.html`

## 2. 섹션4 (How to AI) 내부 그리드선 → 점선 유지

섹션4의 `.step-block` 내부 구분선(수평/수직)은 섹션 간 구분이 아닌 **그리드 내부 선**이므로 점선이 맞음. 하지만 현재 `repeating-linear-gradient` 패턴은 섹션2의 `1px dashed`와 스타일이 다름.

**변경**: 섹션5(Process)의 `.step-block::before/::after`와 `.step-left::after`를 `1px dashed var(--border)`로 통일 (섹션2 era-grid의 점선 스타일과 동일하게).

> 단, 섹션 외곽 테두리(`.section-border-process::before/::after`)는 실선으로 변경.

## 3. 디자인 시스템 통일 제안

### 선(Border) 시스템
```
섹션 외곽 & 구분: 1px solid var(--border)     ← 실선
그리드 내부 분리:  1px dashed var(--border)    ← 점선
```

### 컬러 토큰 정리
현재 `rgba(255,255,255,0.12)`, `rgba(255,255,255,0.08)`, `rgba(255,255,255,0.15)` 등이 혼재. 이미 `--border: #232323`이 정의되어 있으므로:
- 섹션 구분/외곽선: `var(--border)` 통일
- 내부 미세 구분: `var(--border-light)` (이미 `rgba(255,255,255,0.08)` 정의됨)

### 섹션 패딩
현재 섹션별 padding-top/bottom이 80px~140px로 불일치:
- 대형 섹션: `padding: 120px 0` (Section 3, 6, 7, 9)
- 중형 섹션: `padding: 96px 0` (Section 4, 5, 8)
→ 현재 값이 크게 문제되지 않아 **변경하지 않음** (과도한 수정 방지)

### 컨테이너 padding
- `.container`: `padding: 0 120px`
- `.container-howtoai`: `padding: 0 80px`
- `.geo-inner`: `padding: 0 120px`
→ `.container-howtoai`를 `padding: 0 120px`로 통일 검토. 단, 콘텐츠 폭 영향 확인 필요.
